# Реестр SSH-ключей флота (root + vova authorized_keys)

Канонический набор — **4 ключа** на каждой ноде (root и vova). Аудит 2026-07-23: на всём
флоте живут ровно эти 4, чужих/рогов нет. Тела ключей неизменны; комментарии приведены к
чистым уникальным именам (раньше три разных смысла прятались под `vlad@vlad-ubuntu`/PuTTY-именем).

| Имя (комментарий) | Тип | Fingerprint (SHA256, префикс) | Владелец / назначение |
|---|---|---|---|
| `vlad-dm` | RSA | `Irm91dPxi…` | Ключ Vlad'а (`~/.ssh/id_rsa`) — DeployManager, рабочая станция. Он же = FileZilla коллеги. Раньше звался `vlad@vlad-ubuntu` **и** `rsa-key-20220713` (дубль). |
| `servermanager2` | RSA | `tY6Pvu1wc…` | Бот ServerManager2 (PM) — управление сервисами по SSH. Раньше тоже `vlad@vlad-ubuntu` → путаница (инцидент VIDEO-3 07-23). |
| `cluster_root@PRIMARY` | ED25519 | `AwlQHNVdG…` | Автоматизация кластера + терминальный доступ коллеги. |
| `root@primary-pq-f` | RSA | `+G0g7Lxy2…` | Операторский ключ флота. |

## Как поддерживается
- **Новая нода**: `vlad-dm` кладётся провижином (`--vova-pubkey` = `id_rsa.pub`), остальные 3 —
  шагом `operator_access` мастера (из `operator_keys.pub`). Итог — канонический набор автоматом.
- **Выравнивание существующих**: `scratchpad/tidy_keys.sh` (атомарная перезапись per-node,
  гард «не писать без vlad-dm»).
- **Ротация/новый оператор**: добавить строку в `operator_keys.pub` (или сменить тело) + прогнать
  `operator_access`/tidy по флоту + обновить этот реестр.

## Инвариант
Один комментарий = один ключ (fingerprint). Не заводить два разных ключа под одним именем —
именно это скрывало отсутствие ключа SM2 на новых нодах.
